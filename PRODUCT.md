# Product

## Register

product

## Users

The primary user is the owner of this local Jarvis workspace, working at a desktop in Chinese and expecting the assistant to feel present, attentive, and useful throughout the day. The interface is not a marketing site. It is a personal command surface for voice interaction, situational awareness, AI news, tasks, weather, and future skills.

## Product Purpose

Jarvis is a local-first personal AI control room. It should stay visually alive while waiting, listen for wake phrases, answer through DeepSeek, speak responses aloud, and surface useful live context without fake data. Success means the user can open the app, feel that an intelligent sci-fi presence is standing by, and gradually connect real skills to it.

## Brand Personality

Calm, cinematic, intelligent. The tone should feel like a serious machine consciousness rather than a chatbot page: deep blue light, dense particles, controlled motion, low male voice, and restrained interface panels that support the central entity.

## Anti-references

Avoid generic HTML demo aesthetics, predictable code-pattern motion, flat dashboard cards, childish robot faces, fake live data, decorative sci-fi overlays that do not move, and any interface that feels like a template rather than a living assistant. Do not copy Marvel Jarvis directly; use it only as emotional direction.

## Design Principles

1. The entity is the product: the central particle body must feel alive before any panel competes for attention.
2. Real data or no data: weather, AIHot, location, and future skills should show genuine sources or clearly degrade.
3. Voice first: text input is secondary and should appear only when requested.
4. Motion must feel organic: avoid perfectly regular rotations, repeated loops, and obvious code geometry.
5. Panels are instruments: weather, tasks, AIHot, and skills should read like quiet operational readouts around the central consciousness.

## Accessibility & Inclusion

Respect reduced-motion preferences with a lower intensity fallback. Maintain readable Chinese UI text over dark backgrounds. Voice interaction should remain optional through manual controls, and the UI should make microphone state clear.
