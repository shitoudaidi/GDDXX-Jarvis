# Iteration 02 - Control state vocabulary

1. [Design] Generic buttons had no press feedback; a shared active state now confirms the click.
2. [Function] Disabled buttons retained an actionable cursor; disabled controls now communicate that they cannot be used.
3. [Aesthetic] Primary actions did not visually settle on press; the primary surface now darkens slightly.
4. [Aesthetic] Secondary actions had no pressed surface; the secondary surface now gives a restrained inset cue.
5. [Function] News refresh and pause controls lacked a pressed state; their border and text now respond immediately.
6. [Function] Voice provider cards could be keyboard-focused without a strong container cue; focus-within now outlines the active option.
7. [Function] The minimal standby arrow relied on hover for discoverability; keyboard focus now receives a visible ring.
8. [Function] Message actions lacked a pressed state; copy, expand, and replay now share the action feedback vocabulary.
9. [Aesthetic] Command dock controls did not visually confirm a press; the dock now uses a restrained inset border.
10. [Function] Invalid model and URL fields had no shared visual state; native invalid fields now use the error semantic color.
