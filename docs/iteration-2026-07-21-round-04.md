# GDDXX-Jarvis 迭代记录：2026-07-21 / Round 04

时间：19:17-19:21（Asia/Shanghai）

本轮原则：唤醒要提高召回率，但不能通过无限放宽匹配制造误触。

## 已发现并解决的问题（10/10）

1. **唤醒匹配器嵌在大型 React 文件中，无法独立做正反例测试。** 拆为纯函数模块并导出固定测试样本。
2. **别名表存在重复项，规则难以审计。** 清理重复别名并按强匹配、风险匹配、前缀分组。
3. **裸词 `Travis` 会直接唤醒。** 英文风险假设现在必须搭配 Hi、Hey、Hello 或 Yo。
4. **“嘉维是一位同事”等长句会被中文短匹配误触。** 裸同音短词只在真正的短呼叫长度内生效。
5. **早上好、下午好、晚上好未作为唤醒前缀。** 增加三个时间问候前缀。
6. **英语口语前缀 Yo 未覆盖。** 增加 Yo，同时仍要求后接 Jarvis 假设。
7. **繁体中文“賈維斯”未覆盖。** 增加繁体标准写法。
8. **“贾维士、加微斯、佳微斯、嘉微斯、贾薇斯”等常见同音字缺失。** 纳入强匹配集合。
9. **主麦克风启动错误被吞掉。** 保存错误名和消息，并写入 `mic-start-error` 诊断。
10. **指定设备失败后，默认设备回退错误也被吞掉。** 保存回退错误并写入 `mic-fallback-error` 诊断。
